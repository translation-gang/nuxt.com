# Аудит перевода документации (фаза 3.0)

Документация хранится в репозитории **translation-gang/nuxt** (директория `docs/`). Сайт nuxt.com подтягивает её через `content.config.ts`.

## Как провести аудит

1. Клонируйте репозиторий с документацией (если ещё не клонирован):
   ```bash
   git clone https://github.com/translation-gang/nuxt.git ../nuxt
   ```

2. Из корня **nuxt.com** запустите скрипт, указав путь к репозиторию nuxt:
   ```bash
   NUXT_V4_PATH=/absolute/path/to/translation-gang/nuxt node scripts/audit-docs-translation.mjs
   ```
   Либо добавьте в `.env`:
   ```
   NUXT_V4_PATH=/path/to/translation-gang/nuxt
   ```
   и выполните:
   ```bash
   node scripts/audit-docs-translation.mjs
   ```

3. Скрипт перезапишет этот файл отчётом: по каждому разделу список **переведённых** (есть кириллица) и **не переведённых** файлов.

## Интерпретация

- **Переведены** — в файле есть кириллица (хотя бы один символ). Могут потребоваться правки, если upstream обновился.
- **Не переведены** — кириллицы нет, файл нужно перевести (фазы 3.1–3.8).
- После мержа с upstream имеет смысл запускать аудит заново и помечать файлы «переведены, но устарели» для выборочного обновления.

## Ручная проверка (без скрипта)

Из корня репозитория **translation-gang/nuxt**:

```bash
# Все .md с кириллицей (= переведённые)
grep -rl '[а-яА-ЯёЁ]' docs/ --include='*.md' | sort

# Список всех .md в docs/
find docs -name '*.md' | sort
```

Сравните два списка, чтобы получить «не переведённые».
