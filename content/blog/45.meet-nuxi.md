В апреле мы запустили AI-чат на [nuxt.com](/) в [бете](/blog/introducing-nuxt-agent). Он умел многое: знал документацию, искал issues и показывал модули и шаблоны в своих компонентах.

Хорошая демонстрация того, как собрать чат на Nuxt.

Но чат-интерфейсы уже везде, и мы хотели, чтобы Nuxt ощущался иначе. Нужна была своя идентичность агента — в духе бренда Nuxt. И чтобы он стал частью вашего опыта на nuxt.com, а не отдельным виджетом.

Сегодня представляем **Nuxi** — вашего спутника в мире Nuxt.

:agent-nuxi-icon{class="block mx-auto my-6 size-20 text-primary"}

## Зачем Nuxi

Nuxi должен улучшать developer experience и передавать дружелюбие и заботу, за которыми стоит сообщество Nuxt.

Он должен ощущаться как Nuxt.

Поэтому имя **Nuxi**. Раньше так запускали dev-сервер (`nuxi dev`). Сейчас команда `nuxt dev`, но Nuxi остался в памяти команды — и мы вернули его.

::nuxi-mood-gallery
::

## К чему мы стремимся

**Игра.** Nuxi должен быть живым: реагирует на страницу и действия, внутри есть сюрпризы — часть лучше найти самому. Начните с `hi`.

::try-nuxi{prompt="hi" label="Поздороваться"}
::

::tip{icon="i-lucide-mouse-pointer-2"}
Наведите курсор на иконку Nuxi в шапке и подвигайте мышью.
::

**Контекст.** Nuxi знает, на какой странице вы находитесь. Не нужно каждый раз объяснять, о чём речь. Спросите «как использовать это в приложении?» в доке — и «это» уже понятно.

::try-nuxi{prompt="How do I use this in my app?"}
::

**Опора на источники.** Документации по Nuxt много. Nuxi через [MCP-сервер Nuxt](/mcp) ищет ответ там, где он актуален: официальные docs, каталог модулей, шаблоны, changelog. При разборе ошибки сначала смотрит issues в `nuxt`, `nuxt-modules` и `nuxt-content`, и только потом отвечает.

::try-nuxi{prompt="How do I fetch data on the server in Nuxt 4?"}
::

**Рост.** Nuxi — не разовый Q&A. Модули, шаблоны, хостинг и посты блога — карточками; из диалога можно открыть [StackBlitz](https://stackblitz.com); если что-то не так — форма обратной связи. Новую функцию можно добавить PR-ом.

::try-nuxi{prompt="Show me official starter templates"}
::

## Что нового сегодня

Самый большой апдейт с апрельской беты. Цель — сделать [nuxt.com](/) менее шаблонным и ближе к вам.

### Продолжить с того места, где остановились

Теперь можно **войти на nuxt.com** через GitHub. Диалоги сохраняются сами. Начали [чат](/dashboard/chat) на телефоне — закончите на ноутбуке.

::callout{icon="i-lucide-log-in" to="/login"}
Вход через GitHub: [/login](/login).
::

### Ответвления в чате

Диалог не всегда линейный. Нужен отступ в сторону, другой подход или сравнение — разветвите чат. В любом ответе Nuxi нажмите **Ответвить в новый чат**: всё до этого момента копируется в новую ветку, исходный чат не трогается.

::video{poster="https://res.cloudinary.com/nuxt/video/upload/so_0/v1779446023/nuxt/nuxi-branch_jidlol.jpg" controls class="rounded-lg"}
  :source{src="https://res.cloudinary.com/nuxt/video/upload/v1779446023/nuxt/nuxi-branch_jidlol.mp4" type="video/mp4"}
::

### Поделиться ответом

Чаты **приватны по умолчанию**. Можно сделать чат публичным и отправить ссылку. Прочитать смогут все, писать — только вы; приватность можно вернуть в любой момент.

::video{poster="https://res.cloudinary.com/nuxt/video/upload/so_0/v1779446028/nuxt/nuxi-sharing_jwgsxg.jpg" controls class="rounded-lg"}
  :source{src="https://res.cloudinary.com/nuxt/video/upload/v1779446028/nuxt/nuxi-sharing_jwgsxg.mp4" type="video/mp4"}
::

::note
Всё из первого релиза на месте: grounding по docs, карточки модулей и шаблонов, провайдеры деплоя, посты блога, поиск issues на GitHub, ссылки на playground, контекст страницы и **Report issue** с тикетом в Linear. Полный список — в [предыдущем посте](/blog/introducing-nuxt-agent).
::

## Ваше мнение

Nuxt — сообщество, и мы хотим, чтобы то, что строим, было полезно всем. Пишите, что думаете о Nuxi и новых функциях.

::tip{icon="i-lucide-mouse-pointer-2"}
Если ответ Nuxi не устраивает, нажмите **Report issue** в чате. Создастся тикет с полным диалогом — мы читаем каждый.
::

В ближайшие месяцы у [nuxt.com](/) запланировано ещё многое. Будем рады вашим идеям по ходу.

Следите за обновлениями. 👀

## FAQ

::accordion{type="single"}
  ::accordion-item{label="Моя история чатов приватна?"}
  Да. Каждый диалог приватен и виден только вам. Отдельный чат можно сделать публичным в меню видимости в шапке чата и снова закрыть.
  ::
  ::accordion-item{label="Нужен ли вход, чтобы пользоваться Nuxi?"}
  Нет. Nuxi работает без аккаунта. Без входа не будет синхронизации между устройствами, а дневной лимит считается по IP, а не по аккаунту.
  ::
  ::accordion-item{label="Сколько сообщений в день?"}
  Пока двадцать: по аккаунту после входа, по IP без входа. Лимиты подстроим по мере роста нагрузки.
  ::
  ::accordion-item{label="На какой модели работает Nuxi?"}
  Claude Sonnet 4.6 от Anthropic через [Vercel AI Gateway](https://vercel.com/ai-gateway). Grounding — [MCP-сервер Nuxt](/mcp): docs, модули, шаблоны, гайды по деплою и issues на GitHub.
  ::
  ::accordion-item{label="Можно удалить диалог?"}
  Да. Откройте [/dashboard/chat](/dashboard/chat), наведите на чат в боковой панели и удалите через меню. Запись исчезнет из базы.
  ::
  ::accordion-item{label="Что Nuxi знает обо мне после входа?"}
  GitHub username и аватар, плюс ваши диалоги на nuxt.com. Репозитории, код и другие Nuxt-проекты — нет.
  ::
::
